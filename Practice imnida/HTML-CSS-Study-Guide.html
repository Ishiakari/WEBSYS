1. HTML Basics & Document Structure

Every HTML document follows this basic structure:

<!-- Tells the browser this is HTML5 -->
<!DOCTYPE html>

<!-- Root element, lang="en" means English -->
<html lang="en">

<head>
    <!-- Metadata goes here (not visible on page) -->
    <meta charset="UTF-8">
    <title>Page Title</title>
    <link rel="stylesheet" href="styles.css">
</head>

<body>
    <!-- Visible content goes here -->
</body>

</html>

Head Section Elements

Element Purpose
<meta charset="UTF-8"> Character encoding (supports all characters including emojis)
<meta name="viewport"...> Makes page responsive on mobile devices
<title> Text shown in browser tab
<link rel="stylesheet"> Links external CSS file

2. HTML Elements Reference

Container Elements (for grouping)

Element Purpose Semantic?
<div> Generic container for grouping content No
<aside> Sidebar content Yes
<main> Main content area Yes
<nav> Navigation links Yes
<header> Header section Yes
<footer> Footer section Yes
<section> Thematic grouping of content Yes
💡 What is Semantic HTML?
Semantic elements describe their meaning to both browser and developer. Using <nav> instead of <div class="nav"> makes your code more readable and improves accessibility.

Text Elements

Element Purpose
<h1> to <h6> Headings (h1 = biggest, h6 = smallest)
<p> Paragraph
<span> Inline text container (for styling parts of text)
<strong> Bold/important text
<em> Italic/emphasized text

Links & Images

<!-- Link -->
<a href="https://example.com" class="nav-item">Click Me</a>

<!-- Image -->
<img src="photo.jpg" alt="Description" class="avatar">

Form Elements

<input type="text" placeholder="Search...">
<input type="password" placeholder="Password">
<input type="email" placeholder="Email">
<button>Submit</button>

Tables

<table>
    <thead>                    <!-- Table header -->
        <tr>                   <!-- Table row -->
            <th>Name</th>      <!-- Header cell -->
            <th>Email</th>
        </tr>
    </thead>
    <tbody>                    <!-- Table body -->
        <tr>
            <td>John</td>      <!-- Data cell -->
            <td>john@email.com</td>
        </tr>
    </tbody>
</table>

3. CSS Basics & Selectors

CSS Syntax

selector {
    property: value;
    property: value;
}

Types of Selectors

Selector Example What it selects
Universal * All elements
Element p All <p> elements
Class .sidebar Elements with class="sidebar"
ID #header Element with id="header"
Descendant .nav-item span span inside .nav-item
Multiple classes .tab.active Element with both classes
Hover state .btn:hover When mouse is over element
Multiple selectors h1, h2, h3 All h1, h2, and h3 elements
💡 Best Practice
Always use classes (.classname) for styling instead of IDs (#id). IDs should be unique and are better for JavaScript.

4. The Box Model

Every HTML element is a rectangular box with these layers:

┌─────────────────────────────────────────────┐ │ MARGIN │ │ ┌───────────────────────────────────┐ │ │ │ BORDER │ │ │ │ ┌─────────────────────────┐ │ │ │ │ │ PADDING │ │ │ │ │ │ ┌───────────────┐ │ │ │ │ │ │ │ CONTENT │ │ │ │ │ │ │ │ (width x │ │ │ │ │ │ │ │ height) │ │ │ │ │ │ │ └───────────────┘ │ │ │ │ │ └─────────────────────────┘ │ │ │ └───────────────────────────────────┘ │ └─────────────────────────────────────────────┘ 
.element {
    width: 200px;              /* Content width */
    height: 100px;             /* Content height */
    padding: 20px;             /* Space INSIDE the box */
    border: 1px solid #E0E0E0; /* Border around box */
    margin: 10px;              /* Space OUTSIDE the box */
}

Shorthand for Padding & Margin

/* All sides */
padding: 20px;

/* Vertical | Horizontal */
padding: 10px 20px;

/* Top | Horizontal | Bottom */
padding: 10px 20px 15px;

/* Top | Right | Bottom | Left (clockwise) */
padding: 10px 20px 15px 5px;
💡 Box-Sizing Tip
Always add box-sizing: border-box to make width include padding and border. This makes calculations much easier! 
* {
    box-sizing: border-box;
}

5. Flexbox Layout

Flexbox is the most important layout technique for modern CSS. It lets you arrange items in rows or columns easily.

Enable Flexbox

.container {
    display: flex;  /* This enables flexbox */
}

Flex Direction

flex-direction: row;      /* Horizontal (default) → */
flex-direction: column;   /* Vertical ↓ */
flex-direction: row flex-direction: column ┌─────────────────────┐ ┌───────────┐ │ [1] [2] [3] [4] │ │ [1] │ └─────────────────────┘ │ [2] │ │ [3] │ └───────────┘

Main Flexbox Properties

Property Values Purpose
justify-content flex-start, flex-end, center, space-between, space-around Align items on main axis
align-items flex-start, flex-end, center, stretch Align items on cross axis
gap Any length (px, rem, etc.) Space between items
flex-wrap nowrap, wrap Allow items to wrap to next line
flex 1, 2, auto, etc. How much item should grow

Common Flexbox Patterns

/* Center everything */
.center-all {
    display: flex;
    justify-content: center;
    align-items: center;
}

/* Space items evenly */
.space-between {
    display: flex;
    justify-content: space-between;
}

/* Vertical stack with gap */
.vertical-stack {
    display: flex;
    flex-direction: column;
    gap: 10px;
}

/* One item takes remaining space */
.fill-space {
    flex: 1;
}

Typography

.text {
    font-family: 'Poppins', sans-serif;  /* Font with fallback */
    font-size: 14px;                     /* Text size */
    font-weight: 600;                   /* 400=normal, 700=bold */
    line-height: 1.5;                   /* Space between lines */
    text-decoration: none;             /* Remove underline */
    text-align: center;                /* Text alignment */
}

Borders & Rounded Corners

.element {
    border: 1px solid #E0E0E0;   /* width style color */
    border-radius: 8px;          /* Rounded corners */
    border-bottom: 1px solid #F5F5F5; /* Only bottom */
}

/* Make a circle */
.circle {
    border-radius: 50%;
}

Shadows

.card {
    box-shadow: 0 2px 8px rgba(0, 0, 0, 0.1);
}
/*           ↑   ↑   ↑   └─ color with transparency
             │   │   └───── blur radius
             │   └───────── vertical offset
             └───────────── horizontal offset */

Transitions (Smooth Animations)

.button {
    background: #4CAF50;
    transition: all 0.2s ease;  /* Smooth change */
}

.button:hover {
    background: #45a049;  /* This change will be animated */
}

8. Responsive Design

Media queries let you apply different styles based on screen size:

/* Default styles (for large screens) */
.sidebar {
    width: 200px;
}

/* When screen is 768px or smaller */
@media (max-width: 768px) {
    .sidebar {
        width: 70px;  /* Make sidebar smaller */
    }
    
    .nav-item span {
        display: none;  /* Hide text, show only icons */
    }
}

/* When screen is 480px or smaller (mobile) */
@media (max-width: 480px) {
    .sidebar {
        display: none;  /* Hide sidebar completely */
    }
}

Common Breakpoints

Breakpoint Device
max-width: 480px Mobile phones
max-width: 768px Tablets
max-width: 992px Small laptops
max-width: 1200px Large screens
💡 Mobile-First Approach
Many developers prefer writing mobile styles first, then using min-width media queries to add styles for larger screens. This is called "mobile-first" development.
